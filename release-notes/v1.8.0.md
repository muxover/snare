# Snare v1.8.0 — Reverse proxy, grep & full inspection

**Release date:** 2026-05-14

### Install

```bash
go install github.com/muxover/snare@v1.8.0
```

---

## Summary

v1.8.0 adds reverse proxy mode, body rewriting, URL ignoring, remote mapping, memory-only mode, body size capping, a full-text body search command, in-editor replay, selective capture deletion, per-command filter flags, automatic CA trust store installation, colorised output, a protocol column in list/watch, and WebSocket frame display in the TUI.

---

## What's included

**Proxy modes and routing**

- `--mode reverse --target http://backend` — sit in front of a backend without requiring `HTTP_PROXY`.
- `--ignore <substring>` — skip capturing matching URLs; checked at the CONNECT level (whole host) and per request. Repeatable.
- `--map-remote host=http://target` — rewrite the outbound destination for a host. Repeatable.

**Traffic manipulation**

- `--rewrite-body regex=replacement` — transform response bodies with a compiled regexp before capturing and forwarding. Repeatable.

**Capture store**

- `--no-store` — run without writing any files; all captures are held in memory.
- `--max-body-size N` — cap captured body size at N bytes; bodies are truncated with a `truncated` flag, not dropped.

**New commands**

- `snare grep <pattern>` — search all capture bodies (request and response) with a regex. Flags: `--invert`, `--method`, `--host`.

**Command improvements**

- `snare replay --edit` — open the capture JSON in `$EDITOR` before resending. Edited fields (method, URL, headers, body) are used for the outgoing request.
- `snare clear --method/--status/--url/--host` — delete only matching captures. No flags clears everything (previous behaviour).
- `snare watch --method/--status/--url/--host/--body` — filter new captures as they arrive.
- `snare ca install` — executes the platform trust-store command instead of printing instructions.

**Output**

- Protocol column (`h1` / `h2` / `ws`) in `snare list` and `snare watch`.
- Colourised status codes: 2xx green, 3xx yellow, 4xx red, 5xx magenta, errors bold red.
- WebSocket frame list in `snare tui` detail view — direction, timestamp, opcode name, and payload preview (capped at 512 bytes).
