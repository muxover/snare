# Snare v1.4.0 — Mocking & Stubbing

**Release date:** 2026-05-02

### Install

```bash
go install github.com/muxover/snare@v1.4.0
```

## Summary

Adds a full mock/stub system. Define rules that intercept matching requests and return fixed responses — without hitting the real server. Rules persist to disk and load automatically on `serve`.

## What's included

- `snare mock add` — add a mock rule with method, URL pattern, status, body, content-type, and headers
- `snare mock from [id]` — generate a mock from an existing capture's response in one command
- `snare mock list` — list all rules with ID, method, status, and pattern
- `snare mock remove [id]` — remove a rule by ID or prefix
- `snare mock clear` — wipe all rules
- `snare serve --mock-file` — load rules from a custom path at startup
- Rules stored as JSON at `~/.snare/mocks.json` (or `SNARE_MOCKS` env)
- First-match wins; matched requests are never forwarded to the origin
