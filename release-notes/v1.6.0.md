# Snare v1.6.0 — TUI & Scripting

**Release date:** 2026-05-02

### Install

```bash
go install github.com/muxover/snare@v1.6.0
```

## Summary

Adds an interactive terminal UI for browsing captures, plus three scripting/automation primitives that make Snare useful in CI pipelines and shell workflows.

## What's included

- `snare tui` — full-screen terminal UI (Bubble Tea). Live list that refreshes every 2 s, keyboard navigation (↑↓ or j/k), full request/response detail view with scrolling, inline replay with `r`, URL/method filter with `/`.
- `snare pipe` — stream the entire capture store as NDJSON to stdout. `--follow` tails new captures as they arrive. Works naturally with `jq`, `grep`, and any tool that reads JSON.
- `snare assert` — assert that captures match conditions and exit 1 if they don't. Use `--url`, `--method`, `--status`, `--body`, `--min`, and `--max` to build assertions. Designed for CI smoke tests.
- `snare serve --on-capture <cmd>` — hook any shell command onto the proxy. Every new capture is JSON-serialised and written to the command's stdin.
- Release binaries now cover 5 platforms: `linux-amd64`, `linux-arm64`, `darwin-amd64`, `darwin-arm64`, `windows-amd64`.

## Examples

```bash
# stream all 500 errors to jq
snare pipe | jq 'select(.response.status_code==500)'

# tail new captures live
snare pipe --follow | jq -c '{method: .request.method, url: .request.url}'

# CI assertion — at least 1 successful login call
snare assert --url /auth/login --status 200 --min 1

# run a webhook on every capture
snare serve --on-capture 'curl -s -X POST https://hooks.example.com -d @-'
```
