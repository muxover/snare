# Snare v2.2.0 — JS Hooks

**Release date:** 2026-06-26

### Install

```bash
go install github.com/muxover/snare/v2@v2.2.0
```

---

## Summary

v2.2.0 adds a JavaScript hook system. Drop a `.js` file on the command line and snare calls its functions for every request — before forwarding, after the response, and after the capture is saved. No restart needed; the file is reloaded from disk on every request.

---

## What's included

### JS hook files (`--hook`)

```bash
snare serve --hook hook.js
snare serve --hook auth.js --hook logging.js   # multiple files run in order
```

Three optional functions can be defined in each file:

**`onRequest(r)`** — called before the request is forwarded to the origin. Mutate `r.method`, `r.url`, `r.headers`, or `r.body` to change what goes out. Return a response object to short-circuit — snare answers the client locally and never contacts the origin:

```js
function onRequest(r) {
  r.headers["X-Forwarded-By"] = "snare";

  if (r.url.indexOf("/internal/") >= 0) {
    return { status: 403, headers: {}, body: "forbidden" };
  }
}
```

**`onResponse(r, res)`** — called after the response is received. Mutate `res.status`, `res.headers`, or `res.body` to change what the client sees and what is stored in the capture:

```js
function onResponse(r, res) {
  res.headers["X-Proxied"] = "yes";
  res.body = res.body.replace("staging.internal", "production.example.com");
}
```

**`onCapture(c)`** — called after the capture is saved to disk. Read-only view of the full capture JSON, useful for custom logging or alerting:

```js
function onCapture(c) {
  if (c.response && c.response.status_code >= 500) {
    console.log("5xx:", c.request.method, c.request.url);
  }
}
```

`console.log()` writes to snare's verbose log (`-v`). Hook errors are logged and skipped — a broken hook never crashes the proxy. The file is reloaded from disk on every request, so edits take effect immediately without restarting.

### JS engine

The hook system runs on [Goja](https://github.com/dop251/goja), a pure Go ES2015+ JavaScript runtime with no CGO dependency and fast startup. A fresh runtime is created per file per request, so hook files are fully isolated and stateless across requests.
