# Snare v1.9.0 — Chaos, playback & dev tools

**Release date:** 2026-05-15

### Install

```bash
go install github.com/muxover/snare@v1.9.0
```

---

## Summary

v1.9.0 adds chaos engineering and artificial latency to the proxy, a VCR-style record/playback system for offline development, OpenAPI spec generation from captured traffic, Postman collection export, a `curl` command formatter, automatic browser launch, and mobile CA install via ADB and Safari.

---

## What's included

**Chaos and latency**

- `snare serve --delay <duration>` — inject artificial latency before every proxied response (e.g. `--delay 200ms`).
- `snare serve --chaos <pct>` — randomly drop a percentage of requests with 503 to test resilience (e.g. `--chaos 10`).

**Record and playback**

- `snare record` — run the proxy and write all captures to a cassette file (NDJSON, one capture per line).
- `snare playback <cassette>` — serve recorded responses from a cassette file; matches by method and path. No internet needed.

**Developer workflow**

- `snare curl <id>` — print any capture as a ready-to-paste `curl` command with all headers and body.
- `snare export --format postman` — export captures as a Postman Collection v2.1.0 JSON file.
- `snare openapi` — generate an OpenAPI 3.0.3 JSON spec from captured traffic; numeric and UUID path segments become `{id}` automatically.

**Browser integration**

- `snare serve --browser` — auto-launch Chrome, Edge, or Chromium with `--proxy-server=` pointed at the running proxy.

**Mobile CA install**

- `snare ca install --device android` — push the CA to a connected Android device via ADB and install it in the user trust store.
- `snare ca install --device ios` — serve the CA over a temporary local HTTP server so Safari can download and install it directly.
