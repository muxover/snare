## Summary

v2.1.0 adds first-class support for four protocol types that were previously opaque: SSE streams are parsed frame by frame, GraphQL operations are identified and indexed, gRPC frames can be decoded with a `.proto` file, and reverse proxy mode now serves HTTP/3 over QUIC alongside the existing TCP listener.

---

## What's included

### SSE frame capture

Responses with `Content-Type: text/event-stream` are relayed to the client line by line while each parsed SSE event is stored in the capture. Each frame records its `id`, `event`, `data`, and `timestamp`. `snare show` prints the frames after the response section. The TUI detail view and web dashboard both show an SSE frames section. HAR export includes `_sseEvents` on SSE entries.

### GraphQL field extraction

Requests with `Content-Type: application/json` whose body has a top-level `query` key, or requests with `Content-Type: application/graphql`, are detected automatically. The `operation_name`, `operation_type` (query / mutation / subscription), and `variables` are stored under `graphql` in the capture JSON.

`snare list --operation <name>` and `snare watch --operation <name>` filter captures by GraphQL operation name. `snare show` prints the GraphQL section when present. The TUI detail view and web dashboard both surface the operation name and type.

### gRPC protobuf decode

`snare serve --proto <file.proto>` (repeatable) loads a `.proto` file at startup. When a gRPC capture matches a service and method defined in the file, the request and response frames are decoded into human-readable JSON and stored as `grpc.decoded_request` and `grpc.decoded_response`. `snare show`, the TUI detail view, and the web dashboard all display decoded JSON in place of raw bytes when available. If no matching proto is loaded or decoding fails, raw bytes are shown as before.

### HTTP/3 / QUIC

In reverse proxy mode, `snare serve` now starts a QUIC listener on the same UDP port as the TCP listener. The listener uses a cert generated by the MITM CA. Captures from HTTP/3 connections include `protocol: "h3"` and are shown with an h3 badge in `snare list`, the TUI, and the web dashboard. Use `--no-h3` to disable the QUIC listener.

In forward proxy mode, `Alt-Svc` headers are stripped from all proxied responses to prevent clients from directly upgrading to HTTP/3 outside the proxy.
