# Snare v2.4.0 — Collaboration & UX

**Release date:** 2026-07-01

### Install

```bash
go install github.com/muxover/snare/v2@v2.4.0
```

---

## Summary

v2.4.0 makes it easy to share debugging context across machines, adds a smarter JSON diff view, and lets you filter by response time across all commands.

---

## What's included

### `snare bundle` — portable capture bundles

Pack captures, mock rules, and sessions into a single gzip-compressed file and send it to a teammate or commit it alongside your tests.

```bash
# Pack everything
snare bundle pack

# Pack a specific session
snare bundle pack --session checkout-flow --out checkout.snare

# Pack specific captures by ID
snare bundle pack --ids abc123,def456 --out subset.snare

# Unpack on another machine
snare bundle unpack checkout.snare
```

The `.snare` format is gzipped NDJSON — one JSON record per line, one line per capture/mock/session. Readable with `zcat | jq`. On unpack, captures are skipped on ID collision and mocks/sessions are skipped if the name/ID already exists, so it is safe to run multiple times.

`snare export --format bundle` is a shortcut that packs all recent captures into `export.snare`.

### JSON semantic diff

`snare diff <a> <b>` now shows a key-by-key breakdown when both bodies are JSON objects:

```
Response body (differ)
  - old_field: "removed value"
  + new_field: "added value"
  ~ status: "pending" → "complete"
  ~ count: 3 → 4
```

Added keys are green, removed keys are red, changed values are yellow. Non-JSON bodies fall back to the previous side-by-side view.

### `--slow <ms>` filter

Filter by round-trip duration across `list`, `watch`, and `assert`:

```bash
# Show requests that took more than 500ms
snare list --slow 500

# Tail only slow requests live
snare watch --slow 1000

# Fail CI if any request exceeded 200ms
snare assert --slow 200 --max 0
```
