# Snare v2.0.0 — Web UI, TUI overhaul, shadow proxy & gRPC

**Release date:** 2026-05-17

### Install

```bash
go install github.com/muxover/snare/v2@v2.0.0
```

---

## Summary

v2.0.0 is the "complete tool" release: every feature is now reachable from all three modes — CLI, TUI, and browser dashboard. This release adds a full browser UI, a rebuilt TUI with four management tabs, shadow proxy, gRPC frame capture, session diff, and a config file.

---

## What's included

**Web dashboard**

- `snare serve --web` — start a browser dashboard (default port 8080) alongside the proxy.
- Live capture list with SSE push — new requests appear instantly without polling.
- Click any capture to inspect request headers, request body, response headers, response body, gRPC frames, and WebSocket frames.
- Inline replay with editable method, URL, headers, and body. Diff any two captures with pin-and-compare.
- Copy curl command for any capture directly from the browser.
- Export captures as JSON, HAR, Postman collection, or OpenAPI 3.0 spec.
- `--web-port <port>` — choose a different port for the dashboard.

**Web dashboard — Mocks tab**

- Add mock rules (method, URL pattern, status, Content-Type, body) from the browser.
- View all active rules. Remove individual rules or clear all.
- One-click mock from any captured response — pre-fills all fields.

**Web dashboard — Intercept tab**

- Live list of paused requests waiting for a decision.
- Forward, drop, or edit-and-forward each request from the browser.
- Edit method, headers, and body before forwarding.

**Web dashboard — Sessions tab**

- Start and end named sessions from the browser.
- Compare any two sessions side-by-side: shows method, path, and status per request with match/miss highlighting.
- Delete any session directly from the sessions list.

**Web dashboard — CA Cert**

- "CA Cert" button in the nav opens a modal with a direct download link for the CA certificate.
- The modal shows per-device LAN URLs (e.g. `http://192.168.1.x:8080/ca.pem`) for installing on phones that can't reach `127.0.0.1`.
- Step-by-step installation instructions for iOS (Safari) and Android (ADB or browser download).

**Web dashboard — live sync across tabs**

- Clearing or deleting captures from any open tab pushes a named SSE event to all other tabs; the capture list updates everywhere without a reload.
- `snare serve` now auto-opens the dashboard URL in the default browser on startup.

**TUI — four management tabs**

The terminal UI has been fully rebuilt. Press `1`–`4` to switch tabs; all proxy features are accessible without the CLI.

- **Captures** — browse (`↑↓`/`jk`), filter (`/`), inspect (`enter`), replay (`r`), edit & replay (`e`), mock from capture (`m`), delete (`d`), clear all (`C` then confirm), diff two captures (space to mark, `D` to compare).
- **Mocks** — view active rules, add new rules (`a`) with an inline form, delete rules (`d`), clear all (`C`).
- **Intercept** — view paused requests, forward (`f`), drop (`x`), or edit-and-forward (`e`) with an inline edit form.
- **Sessions** — start sessions (`s`), end sessions (`e`), delete sessions (`x`), mark and diff two sessions (space + `D`).
- `snare tui --proxy <url>` — route TUI replays through a proxy so they are captured (default: `http://127.0.0.1:8888`).

**Shadow proxy**

- `snare serve --shadow <url>` — silently duplicate every proxied request to a second target. The duplicate fires asynchronously; the client never waits for it. Repeatable for multiple shadow targets.

**Plugin system**

- `snare serve --plugin <cmd>` — run a shell command for every capture; the full capture JSON is written to its stdin. Multiple `--plugin` flags register independent plugins that all run in parallel.

**gRPC capture**

- Requests with `Content-Type: application/grpc*` are parsed into length-prefixed frames automatically. Frames are stored under `grpc.frames` in the capture JSON with direction (`request`/`response`) and raw payload. The service method (`/package.Service/Method`) is stored under `grpc.method`. Frames are shown in the TUI, CLI `show`, and browser dashboard.

**Replay always captured**

- `snare replay` now routes through the snare proxy by default (`--proxy http://127.0.0.1:8888`) so every replay appears in the capture list and can be inspected. Pass `--proxy ""` to bypass.
- Web dashboard inline replay and TUI replay (`r`) route through the same proxy for the same reason.

**Auto run session**

- `snare serve` automatically creates a named session (`run-YYYY-MM-DDTHH:MM:SS`) on startup and closes it on shutdown. Every proxy run is its own session, visible in `snare session list` and all three UIs.

**Captures pre-loaded on startup**

- When the web dashboard starts, all existing captures on disk are loaded into memory immediately. Prior captures appear in the dashboard and TUI without requiring a manual reload.

**Session management**

- `snare session start <name>` / `snare session end <name>` — bracket a named capture session.
- `snare session list` — list all recorded sessions with timestamps.
- `snare session delete <name>` — delete a named session.
- `snare session diff <a> <b>` — compare two sessions' capture sequences by position: shows mismatched method, path, and status code for each divergent request.

**Config file**

- `~/.snare/config.yaml` — set any `snare serve` flag persistently. CLI flags always take precedence. Suppress with `--no-config`.
