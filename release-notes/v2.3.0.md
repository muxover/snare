# Snare v2.3.0 — Testing & CI

**Release date:** 2026-07-01

### Install

```bash
go install github.com/muxover/snare/v2@v2.3.0
```

---

## Summary

v2.3.0 adds a complete testing and CI layer on top of the existing proxy. You can now write YAML test suites, record golden baselines, check regressions between deploys, and fuzz captured requests — all from the same binary, all with captures visible in the dashboard.

---

## What's included

### `snare test` — YAML suite runner

Define tests in a YAML file and run them against any server:

```yaml
tests:
  - name: health check returns 200
    url: http://localhost:3000/healthz
    method: GET
    expect:
      status: 200
      body_contains: '"ok":true'

  - name: invalid login returns 401
    url: http://localhost:3000/login
    method: POST
    headers:
      Content-Type: application/json
    body: '{"user":"x","pass":"wrong"}'
    expect:
      status: 401
```

```bash
snare test suite.yaml
snare test suite.yaml --proxy http://127.0.0.1:8888   # capture all test traffic
snare test suite.yaml --format junit > results.xml     # GitHub Actions / Jenkins
snare test suite.yaml --format tap                     # TAP output
snare test suite.yaml --parallel                       # run concurrently
```

Exits 0 if all pass, 1 if any fail.

### `snare assert --format junit`

The existing assert command now emits JUnit XML for CI dashboards:

```bash
snare assert --url /api/health --status 200 --format junit > assert.xml
```

Behavior and exit codes are unchanged.

### `snare diff --golden` — regression testing

Record a session as a golden baseline, then check future sessions against it:

```bash
snare session start baseline
# run tests
snare session end baseline
snare diff --golden baseline

# deploy, run tests again
snare diff --check baseline
snare diff --check baseline --strict                         # also compare bodies
snare diff --check baseline --strict --ignore-fields ts,id  # skip volatile fields
```

Exits 1 if any regression is found. Golden files are plain JSON stored in `~/.snare/golden/` — commit them alongside your tests.

### `snare fuzz` — mutation testing

Take any captured request and send N mutated variants to surface obvious gaps:

```bash
snare fuzz <id>
snare fuzz <id> --count 50 --proxy http://127.0.0.1:8888
```

Default mutations: HTTP method cycling, path traversal variants, header removal/corruption, JSON body boundary values. Output table highlights 5xx responses and status-class regressions. Not a security fuzzer — a developer sanity check.
